概率论与数理统计大纲整理

发表于 2020-05-20 更新于 2020-05-15

概率论与数理统计大纲整理

  • 第一章 随机事件及其概率

1.1 随机事件

  • 随机现象：
    在一定条件下无法预知其结果的现象，称为随机现象。
  • 规律统计性：随机现象大量反复出现时所表现的量的规律称为规律统计性。（是随机试验的目的）
  • 随机试验：对随机现象的观察称为随机试验

    其具有三点特性：
    1. 可重复性：试验可以在相同条件下重复运行。
    2. 可观察性：每次试验的结果可能不止一个，并且事先能明确成果的所有可能性。
    3. 不确定性：在进行试验之前不能确定其结果，但必然会出现结果中的一个。
  • 样本空间：将随机试验的每一种可能的结果称为一个样本点，他们的全体称为样本空间，记为$S$（或$\Omega$）
  • 事件：具有某一可观察特性的随机试验的结果称为事件。
  • 随机事件：在试验中可能发生也可能不发生的事件，通常用$A$,$B$,$C$来表示
  • 必然事件：每次试验必然发生的事件。
  • 不可能事件：在任何一次试验中都不可能发生的事件，用空集符号$\emptyset$表示。

1.1.1 事件的集合表示及其运算规律

  1. 包含关系：若$A \subset B$,则称事件$A$包含于事件$B$。其含义是：事件$A$发生必然导致事件$B$发生。
  2. 相等关系：若$A \equiv B$,则称事件$A$与事件$B$相等，其含义是：事件$A$发生必然导致事件$B$发生，反之亦然。即$A \equiv B$，且$B \equiv A$。
  3. 事件$A \cup B$称为事件$A$与事件$B$的和（或者并）。其含义是：当且仅当事件$A$，$B$中至少有一个发生时，事件$A \cup B$发生，有时也记为$A + B$
    类似的，称$\bigcup_{i=1}^n Ai$为n个事件和事件。
  4. 事件$A \cap B$称为事件$A$与事件$B$的积，其含义是：当且仅当事件$A$，$B$同时发生时，事件$A \cap B$发生，$A \cap B$也记作$AB$。与和事件类似都可以视作可数个事件的积事件。
  5. 事件$A-B$称为事件$A$与事件$B$的差，其含义是：当且仅当事件$A$发生，且事件$B$不发生，事件$A-B$发生。

1.2 随机事件的概率

  • 频率及其性质：若在相同条件下进行n次试验，其中事件$A$发生的次数为$r_n(A)$，则称$f_n(A)=\frac{r_n(A)}{n}$为事件$A$发生的频率。
  • 概率及其性质：在相同条件下重复进行n次试验，若事件A发生的频率$f_n(A) = \frac{r_n(A)}{n}$随着试验次数的增大而稳定与某个常数P附件摆动，则称P为事件$A$的概率，记为$P(A)$
  • 概率的公理化定义：非负性，完备性，可列可加性（具有六条性质）

1.3 古典概型与几何概型

  • 古典概型的定义：
    （1）随机试验只有有限个可能的结果；
    （2）每一个结果发生的可能性大小相同。

关于System.out.println(object)

起因：

今天上java课，讲到类和对象的时候，老师演示了直接输出对象的结果，老师将其称为是“地址”，但输出的两个“地址”一个是6位16进制，一个是7位。于是心想这怎么都不可能是jvm内存地址，于是查找了资料。

实验得出：@后面的数字是调用Integer.toHexString(Object.hashcode())的结果。看上去我们似乎已经得到了结论，这也是大部分博客（Stackoverflow，csdn）上的回答：

返回的是对象的哈希值

甚至源码的第一行说明：

Returns a hash code value for the object.

？？？

我们都知道哈希值是根据哈希函数对原始索引的处理结果，目的是为了在查找时更方便的找到的目标元素。在这样的回答中，我们不能够知道其原始索引是什么。对于一个存储在jvm中的对象的索引，我们很容易想到的是对象的内存地址，但真的如此吗？

为什么要哈希值？

上文已经提过，哈希表的本质是为了索引的快速。所以我们也很容易想到，如此的目的是为了方便查找到这个对象。但在jvm中，由于GC机制，对象内存地址实际是无意义的，因为在不同时期，jvm将其处理在不同的内存地址。所以在内存地址无意义的情况下，对象的唯一索引就是hashCode()。回过头来，老师今天并没有说错，这就是java对象的“地址”。只不过不是内存地址。
